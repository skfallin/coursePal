## Key SQL query designs

### Course list/search

```sql
SELECT courses.*, categories.category_name,
       (SELECT COUNT(bookings.booking_id)
        FROM bookings
        WHERE bookings.course_id = courses.course_id) AS total_bookings
FROM courses
INNER JOIN categories ON courses.category_id = categories.category_id
WHERE categories.category_name LIKE ?
   OR courses.name LIKE ?
   OR courses.description LIKE ?
ORDER BY courses.date;
```

### Existing booking check

```sql
SELECT COUNT(*)
FROM bookings
WHERE user_id = ? AND course_id = ?;
```

### Class list

```sql
SELECT users.user_id, users.first_name, users.last_name, users.email, bookings.booking_date
FROM bookings
INNER JOIN users ON users.user_id = bookings.user_id
WHERE bookings.course_id = ?
ORDER BY bookings.booking_date DESC;
```

### Reports

```sql
SELECT courses.course_id, courses.name AS course_name,
       COUNT(bookings.booking_id) AS total_bookings,
       categories.category_name, categories.category_colour
FROM courses
INNER JOIN categories ON courses.category_id = categories.category_id
LEFT JOIN bookings ON bookings.course_id = courses.course_id
GROUP BY courses.course_id, courses.name, categories.category_name, categories.category_colour
ORDER BY total_bookings DESC, courses.date ASC;
```

## Recommendation algorithm pseudocode

```text
Get logged-in user's preferred category IDs.
Get logged-in user's booked course IDs.
Fetch courses running from now to three months ahead.
For each course:
    Count bookings.
    Skip course if bookings >= capacity.
    Skip course if user has already booked it.
    If course category is a user preferred category:
        Set subtitle to "Recommended" and add to preferred list.
    Else:
        Set subtitle to "Starting soon!" and add to other list.
Sort preferred courses by date ascending.
Sort other courses by date ascending.
Merge preferred courses before other courses.
Display first eight courses on the home page.
```

## Responsive design plan

- `@media (max-width: 599px)`: stack header, navigation, home columns, course detail image/details, footer columns, forms, and tables vertically. Set main text/images/content blocks to full width.
- `@media (min-width: 600px)`: use horizontal desktop layout for header, navigation, home content, course details, and footer.
- Course cards use CSS grid with `auto-fit` so cards wrap without horizontal overflow.
